# Security Policy

## Supported Versions

Kami berkomitmen untuk menjaga keamanan proyek PURE. Berikut adalah versi yang saat ini didukung dengan pembaruan keamanan:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Jika Anda menemukan kerentanan keamanan dalam proyek ini, kami sangat menghargai laporan Anda. Silakan ikuti langkah-langkah berikut:

### Cara Melaporkan

1. **Jangan** membuat issue publik untuk kerentanan keamanan
2. Kirim email ke: **[aryont52@gmail.com](mailto:aryont52@gmail.com)**
3. Sertakan informasi berikut:
   - Deskripsi detail tentang kerentanan
   - Langkah-langkah untuk mereproduksi masalah
   - Dampak potensial dari kerentanan
   - Saran perbaikan (jika ada)

### Apa yang Diharapkan

- **Konfirmasi Penerimaan**: Kami akan mengonfirmasi penerimaan laporan Anda dalam 48 jam
- **Penilaian**: Tim kami akan menilai kerentanan dalam 7 hari kerja
- **Pembaruan**: Kami akan memberikan pembaruan reguler tentang progress perbaikan
- **Kredit**: Jika Anda menginginkan, kami akan memberikan kredit atas penemuan Anda setelah masalah diperbaiki

### Kebijakan Pengungkapan

- Kami meminta Anda untuk **tidak mengungkapkan** kerentanan secara publik sampai kami merilis perbaikan
- Kami akan bekerja sama dengan Anda untuk memahami dan mengatasi masalah secepat mungkin
- Setelah perbaikan dirilis, kami akan mengakui kontribusi Anda (jika Anda menginginkannya)

## Best Practices untuk Pengguna

Untuk menjaga keamanan implementasi PURE Anda:

1. **Jangan pernah** commit file yang berisi API keys atau kredensial sensitif
2. Gunakan HTTPS untuk semua koneksi produksi
3. Perbarui dependencies secara berkala
4. Review kode sebelum deployment
5. Gunakan environment variables untuk konfigurasi sensitif
